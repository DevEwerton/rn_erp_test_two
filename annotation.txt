- About Test 2:
O Teste 2 é focado em Formulários, Manipulação de Inputs e Envio de Dados (POST).

Em resumo, os pilares que cobrimos foram:

* Escrita de Dados (POST): Implementamos o envio de dados para uma API externa usando fetch com o método POST, configurando corretamente os headers (application/json) e o body (JSON.stringify).
* Manipulação de Inputs Controlados: Gerenciamento de múltiplos campos através de um Estado Único (objeto client), garantindo o Single Source of Truth (Fonte Única de Verdade).
* Máscaras em Tempo Real: Criação de lógica para máscaras progressivas (CNPJ), utilizando Expressões Regulares (Regex) para formatar o texto enquanto o usuário digita, melhorando a UX.
* Validação de Dados:
    * Campos Obrigatórios: Verificação de preenchimento antes do envio.
    * Regex de E-mail: Validação de formato de e-mail corporativo.
    * Sanitização: Uso de replace(/\D/g, '') para limpar caracteres especiais (pontos, barras, traços) antes de enviar os dados puros para o banco de dados.
* TypeScript Avançado: Uso do operador keyof para criar uma função handleChange genérica e type-safe, evitando erros de digitação nos nomes dos campos do formulário.
* UX e Feedback: Tratamento de fluxos de sucesso (limpeza do formulário com ClientMock) e erro, além de garantir o uso correto de keyboardType (numérico, e-mail) e autoCapitalize.

Em uma frase: O Teste 2 validou sua capacidade de criar interfaces de entrada de dados robustas, garantindo que a informação chegue ao servidor de forma limpa, validada e segura (essencial para módulos de cadastro em ERPs).
